# R208 任務取消補償

## 使用者故事

身為薪資管理人員，
我希望系統能在任務出發前被取消時發放固定補償，
以反映準備成本。

## 驗收條件

- 給定取消前剩餘小時 `cancel_hours_before` 與門檻 `cancel_threshold_hours`。
- 若 `cancel_hours_before <= cancel_threshold_hours`，則給付固定補償 `cancel_compensation_flat`。
- `cancel_threshold_hours` 與 `cancel_compensation_flat` 由規則或參數設定提供。
- `cancel_hours_before` 來源為 `data/missions.json` 的欄位。
